# Minimum Viable Product (MVP)

**Version:** 0.1

**Status:** Draft

**Last Updated:** 2026-06-27

---

## Purpose

The purpose of the MVP is to validate the central hypothesis of ALMA:

Can continuous cognitive reflection help people understand themselves better over time?

The MVP is not intended to demonstrate the full vision of ALMA.

It is designed to test whether the core cognitive architecture provides meaningful value in everyday life.

---

## Core Hypothesis

If ALMA continuously builds and refines a psychological understanding of a person, then users will gain greater self-awareness than through isolated conversations with a general-purpose AI assistant.

---

## MVP Goal

After several weeks of interaction, the user should experience:

* increased psychological clarity;
* recognition of recurring patterns;
* more accurate self-reflection;
* meaningful personal insights.

The objective is understanding rather than advice.

---

## Included Features

The MVP should include only the essential capabilities required to validate the hypothesis.

### Continuous Conversation

The user can return and continue conversations over time.

---

### Cognitive State

ALMA maintains an evolving Cognitive State between conversations.

---

### Observation Generation

Each conversation produces structured psychological observations.

---

### Pattern Recognition

Repeated observations become psychological patterns.

---

### Hypothesis Formation

Patterns generate evolving psychological hypotheses.

---

### Reflection

ALMA periodically shares evidence-based reflections.

---

### Question Generation

ALMA asks follow-up questions that reduce uncertainty.

---

### Personal Growth Timeline

The user can observe how their psychological understanding evolves over time.

---

## Excluded Features

The MVP deliberately excludes:

* partner matching;
* compatibility scoring;
* psychological diagnosis;
* personality typing;
* social networking;
* gamification;
* coaching programs.

These features may become future extensions but are not necessary to validate the core hypothesis.

---

## Success Criteria

The MVP is considered successful if users report that:

* they understand themselves more clearly;
* reflections feel personally meaningful;
* questions stimulate genuine insight;
* understanding improves over multiple conversations.

---

## Long-Term Vision

The MVP is not the destination.

It is the first experimental implementation of ALMA's cognitive architecture.

Its purpose is to validate the principles before expanding the platform.

---

## Closing Principle

The first version of ALMA should not attempt to do everything.

It should do one thing exceptionally well:

Help a person understand themselves more clearly through continuous cognitive reflection.
