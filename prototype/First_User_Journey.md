# First User Journey

**Version:** 0.1

**Status:** Draft

**Last Updated:** 2026-06-27

---

## Purpose

This document describes the experience of a new user during their first interactions with ALMA.

The goal is not simply to introduce the product.

The goal is to begin building an accurate Cognitive State while providing immediate value through reflection and self-understanding.

---

## Design Philosophy

ALMA should never feel like a psychological test.

It should feel like an intelligent conversation that gradually becomes more personal and insightful over time.

Understanding is built through dialogue rather than questionnaires.

---

# Stage 1 — Welcome

The user opens ALMA for the first time.

ALMA briefly explains:

* what it is;
* what it is not;
* how it works;
* how privacy is protected.

The emphasis is placed on understanding rather than judgment.

---

# Stage 2 — First Conversation

The first conversation is completely open.

Rather than asking a predefined questionnaire, ALMA invites the user to talk about whatever currently feels important.

The objective is to collect the first psychological observations.

---

# Stage 3 — Initial Cognitive State

Following the conversation, ALMA creates the first version of the Cognitive State.

At this stage the model is intentionally incomplete.

Uncertainty remains high.

No strong conclusions are presented.

---

# Stage 4 — First Reflection

Rather than offering advice, ALMA shares a small number of evidence-based reflections.

The reflections emphasize curiosity rather than certainty.

The user should feel understood rather than analyzed.

---

# Stage 5 — First Questions

ALMA generates several follow-up questions designed to reduce uncertainty within the Cognitive State.

The user decides whether and when to continue.

No pressure is applied.

---

# Stage 6 — Ongoing Growth

Each new conversation contributes to:

* stronger observations;
* clearer patterns;
* improved hypotheses;
* higher confidence;
* better reflections.

The Cognitive State evolves gradually over weeks and months.

---

## User Experience Principles

Every interaction should make the user feel:

* understood;
* respected;
* emotionally safe;
* intellectually curious;
* free to disagree.

Trust develops through transparency.

---

## Long-Term Goal

The objective of the first user journey is not to create dependence.

It is to establish the foundation for long-term psychological self-understanding.

Each conversation should leave the user slightly more aware than before.

---

## Closing Principle

ALMA is not designed to impress during the first conversation.

It is designed to become increasingly valuable through continuous understanding over time.
